# Livebase v0.1 — Scope and Non-Goals

## v0.1 should do

### 1. Start local-first
The system should run for one person locally without cloud dependence as its core identity.

### 2. Preserve source
Original notes/documents/links should remain traceable. Derived abstractions should not replace source truth.

### 3. Allow progressive structure
The user should not need to build a heavy schema before using the system.
Structure should grow from actual use.

### 4. Support reliable retrieval
The system should help produce better working context than plain folder search or note scavenging.

### 5. Support write-back
Finished work should be able to leave behind durable, correctly placed residue.

---

## v0.1 should not do

### 1. Not enterprise-first
No multi-tenant architecture, admin-heavy setup, or big-team-first identity.

### 2. Not cloud-dependent by default
Cloud sync/integration may come later, but local use should already stand on its own.

### 3. Not mandatory heavy ontology modeling
Do not make users define a full schema before they can benefit.

### 4. Not chat-first identity
AI interaction may help, but the product should not collapse into “chat with your files.”

### 5. Not feature-complete knowledge management
Do not chase every adjacent capability before the core problem is solved.

---

## Bottom line

`v0.1` should prove one thing clearly:

# that a local-first system can keep knowledge more alive than static notes, file search, or thin AI chat shells do.
