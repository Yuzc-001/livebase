# Livebase — Public Surface

## Short description

A local-first durable working context base for human-agent collaboration.

## One-line alt descriptions

- `Keep durable working context alive.`
- `A local context base for callable, connected, source-grounded work.`
- `Livebase keeps knowledge, standards, and task context from decaying into static notes.`

## Tagline candidates

- `Keep durable working context alive.`

## Public messaging rules

### Emphasize
- working-context decay as the enemy
- local-first core
- source grounding
- callable retrieval and context packs
- durable write-back
- progressive structure, not heavy schema theater
- human-agent shared continuity

### Avoid overemphasizing
- generic AI memory language
- note-taking app comparisons as the main frame
- “chat with your files” framing
- enterprise feature checklists
- abstract second-brain rhetoric without operational meaning

## Core explanation

Livebase is not trying to win by storing more notes.
It is trying to reduce what usually happens after capture:

- things sink
- standards drift
- context breaks
- finished work bypasses the system
- the whole base gets noisier over time

Livebase exists to make a local working context base stay more usable, more connected, and more trustworthy as it grows.
