# Progressive Structure

Use this file when the main question is:

# how much structure should be added now?

## Core rule

Add the lightest structure that materially improves future retrieval, linkage, or write-back.
Do not require a heavy schema before the system becomes useful.

## Prefer
- simple linking
- stable identities for recurring things
- lightweight metadata
- gradual promotion from loose note → durable object when needed

## Avoid
- forcing full ontology design up front
- schema theater
- adding structure with no retrieval or reuse payoff
