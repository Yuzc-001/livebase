# Write-Back

Use this file when the main question is:

# what should return to the system after meaningful work ends?

## Core rule

Finished work should leave behind the smallest durable residue that strengthens future work.

## Possible write-back forms
- next-time rule
- linked note update
- project-state update
- source-linked lesson
- context-pack seed for future work

## Avoid
- ceremonial documentation
- duplicate summaries with no future use
- residue that cannot be traced back to source or context
