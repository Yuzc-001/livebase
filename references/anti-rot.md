# Anti-Rot

Use this file when the main question is:

# why is the system becoming less trustworthy over time?

## Core rule

The system should get stronger with use, not heavier and less reliable.

## Common rot patterns
- duplicate notes
- stale abstractions
- broken links between related objects
- orphaned project context
- AI-generated residue without grounding

## Anti-rot direction

Prefer:
- consolidation where duplication harms retrieval
- source-preserving updates
- stable identities for recurring objects
- small corrective write-backs instead of giant reorganizations
