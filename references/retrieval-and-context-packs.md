# Retrieval and Context Packs

Use this file when the main question is:

# how do we make knowledge more callable than plain note search?

## Core rule

Retrieval should produce dependable working context, not just fragments.

## Good retrieval should preserve
- source grounding
- surrounding context
- connected objects when relevant
- current relevance to the task

## Context pack purpose

A context pack should be the smallest reliable bundle that helps a human or agent continue real work.

## Failure signs

- search finds isolated snippets with no working context
- the same question yields unstable grounding
- users still need to manually reconstruct the thread from many notes
